# Portfolio UI Redesign Skill

專門處理個人網站 UI 重構、首頁聚焦、時間軸美化、資訊密度控制的 Claude Code skill。

## 核心設計原則

### 1. 內容減量優先
- **刪除勝過添加**：每次改版先思考能刪掉什麼，而不是加什麼新元件
- **資訊分層**：核心訊息、支撐訊息、裝飾訊息要有明確層級
- **一個頁面一個重點**：避免平均分配視覺注意力

### 2. 反 Dashboard 設計
- **拒絕卡片堆疊**：若頁面出現 dashboard 感，必須重構
- **避免網格思維**：不要讓所有元件都擺在同等重要的位置
- **打破對齊**：適度的不對齊創造視覺流動性

### 3. 頁面結構要求
- **Clear Hero**：主視覺區域必須佔據足夠空間，傳達核心價值
- **Clear Secondary Section**：次要區域不能搶奪主區域風頭
- **Clear CTA**：行動呼籲要明確但不突兀，位置精心安排

### 4. Timeline 美化規範
- **流動感**：避免死板的垂直或水平排列
- **錯位節奏**：卡片位置要有韻律感，避免機械對齊
- **主次差異**：重要節點與一般節點在視覺上要有明顯差異
- **空間呼吸**：充足的留白讓內容有呼吸空間

### 5. 文案品牌化
- **縮短優先**：所有文案都要縮短成品牌語言
- **避免說明書文案**：用情感化、個性化語言替代功能性描述
- **一致性 Tone**：保持整體語調統一

### 6. 設計系統統一性
- **風格選擇**：白底與深色風格混搭時，優先統一成同一設計系統
- **色彩延續**：確保色彩在不同區塊間有邏輯連貫性
- **組件一致性**：按鈕、卡片、文字樣式要遵循統一規範

### 7. 色彩層級管理
- **避免同亮度**：所有卡片不能有同樣亮度，要有主次之分
- **漸層運用**：適度使用漸層創造深度感
- **重點色運用**：重點色只用在最重要的元素上

### 8. 視覺權重分配
- **Hero 絕對主導**：services / metrics / CTA 一律不准搶 Hero 主視覺
- **次要資訊後退**：支撐資訊要在視覺上退到背景
- **引導視線流**：設計要引導用戶視線有序移動

### 9. 互動設計克制
- **精緻 Hover**：hover 效果要克制、精緻，避免廉價炫技
- **Motion 意義**：每個動畫都要有明確目的，不為動而動
- **載入體驗**：注重載入狀態和過渡動畫的品質

### 10. 改版前檢查清單
每次改版前必須確認：
- ✅ 是否讓頁面更短？
- ✅ 是否讓頁面更清楚？
- ✅ 是否讓頁面更高級？
- ✅ 是否突出了最重要的訊息？
- ✅ 是否移除了不必要的元素？

## 常見問題解決方案

### Hero 區域設計
```
❌ 避免：資訊密度過高、多個 CTA、複雜佈局
✅ 推薦：單一核心訊息、一個主要 CTA、充足留白
```

### Timeline 設計
```
❌ 避免：直線排列、相同尺寸卡片、機械對齊
✅ 推薦：曲線路徑、大小錯落、自然流動
```

### 色彩運用
```
❌ 避免：彩虹色卡片、過度飽和、無層次感
✅ 推薦：單色系漸層、明確主色、留白平衡
```

## 實作指引

### 重構步驟
1. **審視現狀**：識別 dashboard 元素和視覺雜訊
2. **內容梳理**：將內容按重要性分為 3 層
3. **佈局簡化**：移除非必要元件和裝飾
4. **視覺層級**：建立清晰的視覺優先級
5. **精緻打磨**：細調間距、色彩、動效

### 技術實現重點
- 使用 Framer Motion 實現流暢過渡
- Tailwind CSS 的 spacing system 保持一致性
- 響應式設計優先考慮移動端體驗
- 性能優化：減少不必要的重繪和回流

## 品質標準

最終交付的設計必須達到：
- **視覺衝擊力**：3 秒內傳達核心價值
- **資訊清晰度**：用戶能快速找到想要的資訊
- **品牌一致性**：所有元素都服務於整體品牌形象
- **技術穩定性**：在各種設備和瀏覽器上表現一致

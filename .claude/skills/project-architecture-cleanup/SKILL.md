# Project Architecture Cleanup Skill

專門處理專案架構整理、檔案清理、分層重構的 Claude Code skill。

## 核心原則

### 1. 刪除優於重構
- **無用檔案優先刪除**：過時的組件、未使用的工具檔案、重複的邏輯
- **合併相似功能**：多個小組件合併成一個，減少檔案碎片化
- **移除開發遺留**：測試檔案、範例代碼、註釋掉的程式碼

### 2. 分層架構清晰
- **按功能分層**：UI 層 / 業務邏輯層 / 數據層 要有明確邊界
- **避免循環依賴**：組件之間的引用關係要是單向的樹狀結構
- **統一命名規範**：檔案名、組件名、函數名要有一致的模式

### 3. 目錄結構最佳化
- **扁平化優先**：避免過深的嵌套目錄，2-3 層最佳
- **功能導向分組**：按業務功能而非技術類型組織檔案
- **共用組件獨立**：真正被多處使用的組件才放入 shared/common

### 4. 依賴關係優化
- **減少依賴數量**：移除未使用的 npm packages
- **版本統一管理**：同功能的庫只保留一個
- **懶加載實現**：大型組件和頁面要動態載入

## 架構清理檢查項目

### 檔案層級清理
1. **移除未使用檔案**
   - 搜尋無引用的組件檔案
   - 刪除空的或僅有註釋的檔案
   - 移除過時的配置檔案

2. **合併重複邏輯**
   - 相似功能的組件合併
   - 重複的工具函數統一
   - 樣式檔案去重

3. **清理開發痕跡**
   - 移除 console.log
   - 刪除註釋掉的程式碼
   - 移除測試用的假資料

### 目錄結構重構
1. **扁平化目錄**
   ```
   ❌ 避免：
   src/
     components/
       ui/
         common/
           shared/
             Button.js
   
   ✅ 推薦：
   src/
     components/
       Button.js
   ```

2. **功能導向組織**
   ```
   ❌ 技術導向：
   src/
     components/
     hooks/
     utils/
     styles/
   
   ✅ 功能導向：
   src/
     features/
       auth/
       dashboard/
       profile/
     shared/
       components/
       utils/
   ```

### 依賴關係清理
1. **package.json 優化**
   - 移除未使用的依賴
   - 將 devDependencies 正確分類
   - 檢查版本衝突

2. **import 語句整理**
   - 移除未使用的 import
   - 統一 import 順序（第三方 → 本地）
   - 使用絕對路徑替代相對路徑

### 程式碼品質提升
1. **組件職責單一化**
   - 大型組件拆分成小組件
   - 邏輯與 UI 分離
   - 避免一個檔案做太多事

2. **配置集中化**
   - 環境變數統一管理
   - 常數定義集中
   - 主題色彩系統化

## 重構執行步驟

### Phase 1: 分析現狀
1. **依賴分析**：檢查 package.json 和實際使用情況
2. **檔案掃描**：找出未使用、重複、過大的檔案
3. **引用關係**：梳理組件間的依賴關係圖

### Phase 2: 清理冗餘
1. **移除未使用依賴**：npm ls、bundle analyzer
2. **刪除冗餘檔案**：orphaned files、dead code
3. **合併相似組件**：功能重疊的組件整合

### Phase 3: 重構架構
1. **目錄重組**：按新的分層邏輯移動檔案
2. **import 路徑更新**：使用 alias 簡化引用
3. **組件拆分**：大型組件分解成小組件

### Phase 4: 優化配置
1. **工具配置統一**：ESLint、Prettier、TypeScript
2. **打包優化**：code splitting、tree shaking
3. **路徑別名設定**：@/ 簡化 import

## 質量檢查標準

### 檔案組織質量
- ✅ 每個檔案職責明確，不超過 200 行
- ✅ 目錄層級不超過 3 層
- ✅ 檔案命名遵循統一規範
- ✅ 無孤立檔案（沒有被引用的檔案）

### 依賴關係質量
- ✅ 無循環依賴
- ✅ 依賴樹清晰（可視化後容易理解）
- ✅ 共用組件被至少 2 個地方使用
- ✅ 第三方依賴數量合理（<50 個）

### 程式碼質量
- ✅ 無 dead code
- ✅ 無重複邏輯
- ✅ import 語句整潔
- ✅ 配置集中化管理

## 維護原則

### 持續清理
- **定期檢查**：每個 sprint 檢查新增的冗餘
- **重構小步驟**：避免大規模重構，小步迭代
- **文檔同步**：架構變更要更新 CLAUDE.md

### 防止回歸
- **Code Review 檢查點**：新增檔案是否必要
- **自動化檢查**：eslint rules 防止 dead code
- **依賴管控**：新依賴需要 justification

## 常見問題解決

### 複雜重構場景
1. **大型組件分解**：先抽取邏輯，再分離 UI
2. **循環依賴解除**：引入中間層或事件系統
3. **遺留代碼處理**：逐步替換，避免破壞性變更

### 工具建議
- **檔案分析**：webpack-bundle-analyzer
- **依賴檢查**：depcheck、npm-check-updates
- **代碼分析**：ESLint、SonarQube
- **架構可視化**：madge、dependency-cruiser
